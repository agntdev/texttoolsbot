# fix-3f63e32e81086a5a — Command collision: /count registered by both E2T1.ts and E3T1.ts with different output formats

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 TXTBOT

`src/handlers/E2T1.ts:7` and `src/handlers/E3T1.ts:7` both register `composer.command("count", ...)` with different reply formats:
- E2T1 replies `"Words: {words}, Characters: {chars}"`
- E3T1 replies `"Word count: {count}"`

The handler loader in `src/bot.ts:39` iterates files in **sorted order** and registers each via `bot.use()`. Alphabetically, `E2T1.ts` sorts before `E3T1.ts`, so grammY matches E2T1 first and E3T1 is **dead code** — its handler will never fire.

**Consequence:** the E3T1 test spec (`tests/specs/E3T1.json`) expects `"Word count: 2"` but the actual response will be `"Words: 2, Characters: 11"` — **the spec would fail against the real bot**. E2T1 is the superset feature, so E3T1.ts should be removed.
